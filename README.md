## How to Contribute

- **Branching**
  - `main`: stable, review-ready code
  - `feature/<name>`: feature development
  - `fix/<name>`: bug fixes

- **Pull Requests**
  - Create a PR for meaningful changes
  - Include a clear description of what was changed and why
  - Review by at least one team member is recommended

- **Commit Messages**
  - `feat:` new features
  - `fix:` bug fixes
  - `docs:` documentation changes
  - `chore:` maintenance or setup tasks

- **Issues**
  - Use Issues to track tasks, bugs, or discussion points
  - Assign issues when possible to clarify responsibility
